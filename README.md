# 🎞️ Wandercut — 여행 영상 자동 메이커

여행 다녀와서 영상 편집하기 귀찮을 때, **사진과 영상만 넣으면** 촬영 시간순으로
자동 정렬하고 노래의 **BPM에 맞춰 전환**되는 감성 여행 영상을 만들어 주는
Chrome 확장 프로그램입니다. 모든 처리는 **브라우저 안에서** 일어나며, 사진·영상이
외부로 업로드되지 않습니다.

## ✨ 핵심 기능

- **촬영 시간순 자동 정렬** — JPEG의 EXIF `DateTimeOriginal`, MP4/MOV의 `mvhd`
  생성 시각을 직접 읽어 여행 흐름대로 자동 배치합니다. (메타데이터가 없으면 파일
  수정 시간으로 추정하고 "추정" 배지를 표시합니다.)
- **영상 자동 트림** — 영상은 최대 길이(기본 5초)에 맞춰 가운데 구간을 잘라 배치하고,
  사진은 지정한 시간(기본 3초)만큼 노출합니다.
- **BPM 비트 싱크** — 노래를 함께 넣으면 BPM을 분석해 사진·영상 전환을 비트에 맞추고,
  컷 순간에 짧은 플래시와 줌 펀치로 리듬감을 더합니다.
- **Ken Burns 효과 & 크로스페이드** — 사진에는 부드러운 줌/팬, 장면 사이에는
  크로스페이드(또는 컷)를 적용합니다.
- **상용 수준 UI** — 드래그&드롭, 순서 재배치, 실시간 미리보기, 비율(16:9·9:16·1:1)·
  화질(HD·FHD) 선택, 진행률 표시, 결과 미리보기 및 다운로드.

## 📦 설치 (개발자 모드 / 압축 해제된 확장)

1. Chrome 주소창에 `chrome://extensions` 입력
2. 오른쪽 위 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램을 로드합니다** 클릭
4. 이 저장소 폴더 선택
5. 툴바의 Wandercut 아이콘 클릭 → **에디터 열기**

> 빌드 단계가 없습니다. 순수 HTML/CSS/JS(ES 모듈)로 작성되어 바로 로드됩니다.

## 🚀 사용법

1. 에디터에 사진/영상을 끌어다 놓습니다. → 촬영 시간순으로 자동 정렬됩니다.
2. (선택) **배경 음악**에 노래를 넣으면 BPM이 분석됩니다. **BPM 비트 싱크**를 켜세요.
3. 사진 노출 시간, 영상 최대 길이, 화면 비율, 화질, 전환 효과를 조정합니다.
4. **미리보기**로 확인한 뒤 **영상 만들기**를 누릅니다.
5. 완성되면 미리보기에서 재생하고 **다운로드**(`.webm`)합니다.

## 🧠 동작 원리

| 단계 | 모듈 | 설명 |
|------|------|------|
| 메타데이터 추출 | `src/metadata.js` | EXIF/`mvhd` 파싱으로 촬영 시각·방향 추출 |
| BPM 분석 | `src/bpm.js` | 오프라인 저역 통과 → 피크 검출 → 간격 히스토그램 |
| 타임라인 구성 | `src/timeline.js` | 길이 산정·비트 스냅·전환/효과 배정 |
| 렌더링 | `src/renderer.js` | Canvas 실시간 합성 + MediaRecorder 녹화 |

## 🧪 테스트

핵심 로직(타임라인 구성·BPM 분석·메타데이터 파싱·코덱 선택)은 **의존성 없이**
Node 내장 테스트 러너로 검증합니다. 별도 설치 없이 바로 실행됩니다.

```bash
node --test          # 또는 npm test
node --test --watch  # 변경 감지 실행
```

| 테스트 | 대상 | 검증 내용 |
|--------|------|-----------|
| `tests/timeline.test.js` | `buildTimeline` | 길이 산정·트림·비트 스냅(최소 4비트)·전환/효과 배정 |
| `tests/bpm.test.js` | `analyzeAudio` 외 | 피크 검출, 옥타브 폴딩, 합성 클릭 트랙 BPM 추정 |
| `tests/metadata.test.js` | `parseExif`/`parseMp4Date` | 손으로 합성한 EXIF·`mvhd` 바이트로 시각/방향 파싱 |
| `tests/renderer.test.js` | `pickMime` | 코덱 지원 여부에 따른 MIME 폴백 |

브라우저 API(Web Audio·FileReader·MediaRecorder)는 `tests/helpers/fixtures.js`의
최소 모의로 대체하며, 파서는 실제 바이너리 입력으로 테스트합니다.
`.github/workflows/test.yml`이 push/PR마다 동일하게 실행합니다.

## ⚠️ 참고 사항

- 렌더링은 **영상 길이만큼 실시간**으로 진행됩니다(브라우저 내 합성). 탭을 켠 채로
  기다려 주세요.
- 출력 형식은 `.webm`(VP9/VP8 + Opus)입니다. 대부분의 플레이어와 SNS 업로드에서
  재생됩니다.
- HEIC 등 Chrome이 디코딩하지 못하는 이미지 포맷은 메타데이터만 읽히고 화면에는
  표시되지 않을 수 있습니다. JPG/PNG, MP4/MOV 사용을 권장합니다.

```

editor.js           # 오케스트레이터

src/bpm.js          # BPM 분석
src/timeline.js     # 타임라인 빌더
src/renderer.js     # 캔버스 렌더 + 녹화 엔진
icons/              # 확장 아이콘
gen_icons.py        # 아이콘 생성 스크립트(개발용)
tests/              # node:test 단위 테스트 + 바이너리 픽스처
package.json        # 테스트 스크립트(npm test) / type=module
```
